## 项目简介

这是一个专为财务人员设计的库存收发存数据分析工具，用于月底与各 Site 核对材料采购、耗用和结存情况。工具基于 Odoo 系统导出的收发存报表数据源，提供快速筛选、查看、分组、排序功能，并支持按 Site 导出对账明细和 Word 分析报告。

---

## 主要功能

### 1. 数据读取与展示
- ✅ 支持 Excel 文件 (.xlsx, .xls) 读取
- ✅ 自动识别收发存数据列（期初、入库、出库、期末）
- ✅ 智能识别仓库 (Location) 和产品类别 (Category) 列
- ✅ 表格支持列排序（点击表头即可升序/降序）

### 2. 多维度筛选
#### Location 筛选
- 支持按仓库筛选：BOS, HUO, SFO, SAN, LAX, 其他
- 支持多选和全选/全不选功能
- 实时显示筛选后的行数统计

#### 业务类型筛选
- **月初**: 查看期初存货数据
- **采购**: 查看本月采购入库数据
- **耗用**: 查看本月出库耗用数据
- **结存**: 查看期末结存数据

### 3. 数据分组与展示
- ✅ 支持按产品类别分组显示
- ✅ 支持展开/折叠所有分组
- ✅ 自动计算并显示各类别合计
- ✅ 分组显示时展示产品名称和数量统计

### 4. 金额汇总
实时显示四大金额指标：
- 期初金额 (Opening)
- 采购金额 (In)
- 耗用金额 (Out)
- 结存金额 (Ending)

### 5. 整体概览
提供独立窗口展示：
- **按类别汇总**: 各产品类别的期初、采购、耗用、结存金额
- **按 Site 汇总**: 各仓库的期初、采购、耗用、结存金额
- 支持排序和对比分析

### 6. 数据导出
#### Excel 对账明细导出
- 导出格式：多 Sheet Excel 文件
- 包含四个工作表：期初、本期采购、本期耗用、本期结存
- 自动应用 Location 筛选条件
- 自动调整列宽和添加合计行
- 文件命名：`{Site 名称}-stock-balance-detail_{时间戳}.xlsx`

#### Word 分析报告导出
- 专业的财务报告格式，符合国际财务标准
- **报告结构**:
  - **标题**: Monthly Inventory Movement Analysis（居中显示）
  - **称呼**: Dear XXX Site Head（XXX 为所选仓库名称）
  - **开场白**: Please find below the monthly inventory movement analysis for your review.
  
- **报告内容**:
  1. **Summary（汇总）**: 按类别展示期初、采购、耗用、结存的汇总表格
     - 表头：Category, Opening, Purchase, Consumption, Ending
     - 第一列（Category）占 50% 宽度，便于显示完整类别名称
     - 其余列平均分配剩余宽度
  
  2. **Inventory Receipt Analysis（入库分析）**:
     - 显示当月入库总金额和总数量
     - Top 5 入库金额物品明细表格
     - 列包括：产品名称、入库金额列、入库数量列（使用原始 Excel 列名）
  
  3. **Inventory Consumption Analysis（出库分析）**:
     - 显示当月出库总金额和总数量
     - Top 5 出库金额物品明细表格
     - 列包括：产品名称、出库金额列、出库数量列（使用原始 Excel 列名）
  
  4. **Month-End Inventory Balance（期末库存）**:
     - 显示期末总库存价值和总数量
     - Top 5 期末结存金额物品明细表格
     - 列包括：产品名称、结存金额列、结存数量列（使用原始 Excel 列名）
  
  5. **确认事项**:
     - All procurement receipts have been recorded accurately.
     - All consumption transactions have been properly registered.
     - The month-end inventory quantities are consistent with inventory counts.

- **表格特性**:
  - 第一列（产品名称）占 50% 宽度，确保完整显示
  - 其余列（金额、数量）平均分配剩余宽度
  - 使用原始 Excel 文件的列名，保持数据一致性
  - 数值格式化为带千分位的两位小数（如：$1,234.56）
  - 表头加粗显示，使用网格样式
  
- **文件命名**: `{Site 名称}-stock-report_{时间戳}.docx`
- **自动清理**: 自动删除旧的同类报告文件

#### 导出文件夹管理
- 一键打开导出文件夹
- 自动清理旧的同名导出文件

### 环境要求
- Python 3.8 或更高版本
- Windows 操作系统

### 安装依赖

```bash
pip install pandas openpyxl python-docx
```

或安装所有依赖：

```bash
pip install -r requirements.txt
```

### 依赖包说明
- `tkinter`: Python 内置 GUI 库（通常随 Python 自动安装）

- `openpyxl`: Excel 文件读写和样式处理
- `python-docx`: Word 文档生成

## 使用方法

### 启动程序

```bash
python stock_analysis.py
```

### 操作步骤

1. **加载数据**
   - 点击"选择并读取"按钮
   - 选择从 Odoo 导出的收发存汇总表 Excel 文件
   - 程序自动读取并显示数据

2. **筛选数据**
   - 使用 Location 复选框筛选仓库
   - 点击"全选"/"全不选"快速切换
   - 使用功能按钮查看不同业务类型数据

3. **查看数据**
   - 点击表头进行排序
   - 使用"展开"/"折叠"按钮查看分组详情
   - 勾选/取消"分组"复选框切换显示模式
   - 点击"整体概览"查看汇总分析

4. **导出数据**
   - 筛选目标数据后，点击"excel"或"word"按钮
   - 导出文件保存在源文件目录下的"导出"文件夹
   - 点击"目录"按钮快速打开导出文件夹

## 常见问题

### Q: 程序界面显示不全？
**A**: 请确保屏幕分辨率至少为 1920x1080。程序窗口支持调整大小，可以手动拉大窗口。

### Q: 无法识别仓库或类别列？
**A**: 确保 Excel 文件包含"仓库"、"location"、"类别"或"category"等关键词的列名。

### Q: 导出文件为空？
**A**: 请先筛选数据，确保当前显示的数据不为空后再导出。

### Q: 金额汇总显示为$0.00？
**A**: 检查数据列是否包含数值数据，或列名是否符合识别规则。
